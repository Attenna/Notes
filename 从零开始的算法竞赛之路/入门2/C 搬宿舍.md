# 搬寝室・DP 备考笔记

## 一、题目核心

- 给定 n 个物品重量，选 **2k 件** 分成 **k 对**
- 每对代价：(重量差)²
- 求：总疲劳度 **最小**
- 数据范围：n <2000，O (nk) DP 可过

---

## 二、关键结论（必记）

1. **必须先排序**
    
    最小差平方配对 → 一定是**相邻配对**最优。
2. **不能贪心从头到尾两两配对**
    
    局部最优 ≠ 全局最优，必须用 DP。

---

## 三、动态规划设计

### 1. 状态定义

> `dp[i][j]` = 前 **i** 个物品，选出 **j** 对，最小总疲劳度

### 2. 转移方程

对第 i 个物品只有两种决策：

1. **不选第 i 个**
    
    `dp[i][j] = dp[i-1][j]`
2. **选第 i 个**，则必须和 i-1 配对
    
    `dp[i][j] = dp[i-2][j-1] + (w[i]-w[i-1])²`

最终：

dp[i][j]=min(dp[i−1][j], dp[i−2][j−1]+(wi​−wi−1​)2)

### 3. 初始化

- `dp[i][0] = 0`：选 0 对代价为 0
- 其余 `dp[i][j] = INF`（无穷大，表示不可达）

### 4. 答案

`dp[n][k]`

---

## 四、时间 / 空间复杂度

- 时间：**O(n log n + n·k)**
    
    排序 O (n log n) + DP O (nk)
- 空间：**O(n·k)**
    
    可滚动数组优化到 O (k)，比赛一般直接开二维更稳

---

## 五、标准模板代码（C++）

cpp

运行

```
#include <iostream>
#include <algorithm>
#include <cstring>
using namespace std;

const int MAXN = 2005;
const int INF = 1e9;

int w[MAXN];
int dp[MAXN][MAXN];

int main()
{
    int n, k;
    while (cin >> n >> k)
    {
        for (int i = 1; i <= n; i++)
            cin >> w[i];
        sort(w + 1, w + n + 1);

        // 初始化
        for (int i = 0; i <= n; i++)
        {
            for (int j = 0; j <= k; j++)
                dp[i][j] = INF;
            dp[i][0] = 0;
        }

        // DP
        for (int i = 2; i <= n; i++)
            for (int j = 1; j <= k; j++)
                dp[i][j] = min(dp[i-1][j],
                               dp[i-2][j-1] + (w[i]-w[i-1])*(w[i]-w[i-1]));

        cout << dp[n][k] << endl;
    }
    return 0;
}
```

---

## 六、易错点（考试必避坑）

1. **没排序直接 DP** → 答案必错
2. **数组从 0 开始 / 从 1 开始混乱** → 越界 / WA
3. **初始化没设 INF** → 用了非法状态导致答案偏小
4. **j 循环上限写错** → 超时或越界
5. **多组输入不清空数组** → 上一组数据污染下一组

---

## 七、同类题识别（一眼看出是这模型）

看到这些描述直接套这个 DP：

- 从 n 个里选 k 对
- 两两配对，代价与差相关
- 求最小总代价
- n 在 1000~2000 量级

典型模型：**线性 DP + 相邻配对 + 选或不选**

# 为什么 DP 是二维？为什么第二维是 k？

---

## 一句话结论

- **第一维 i**：表示**处理到第几个物品**
- **第二维 j**：表示**已经凑成了几对**
    
    我们要的是：**n 个物品，凑 k 对的最小代价**
    
    所以必须用 **dp[i][j]** 这种二维状态。

---

# 1. 为什么不能只用一维 dp [i]？

如果只开一维：

`dp[i] = 前 i 个物品的最小代价`

那会出现一个致命问题：

**你不知道已经选了多少对！**

比如：

- 前 5 个物品，可能凑了 1 对
- 也可能凑了 2 对
    
    它们的代价完全不一样，未来转移也不一样。

一维数组**存不下 “选了几对” 这个信息**，所以必须再加一维专门记录**对数**。

---

# 2. 第二维为什么是 k（代表 “几对”）？

题目要求非常明确：

> 必须搬 **2k 件物品，分成 k 对**

我们的目标就是：

> 从 n 个里，**恰好选 k 对**，求最小总疲劳。

所以 DP 状态里必须记录：

**现在已经选了 j 对**

j 从 0 到 k。

这就是为什么第二维是 k。

---

# 3. 状态含义再讲一遍（务必背下来）

plaintext

```
dp[i][j]
= 前 i 个物品
= 选出 j 对
= 最小总疲劳度
```

两个维度各司其职：

1. **i（第一维）**：控制**物品范围**
2. **j（第二维）**：控制**配对数量**

---

# 4. 为什么这样设计就能转移？

看转移就懂了：

cpp

运行

```
dp[i][j] = min(
    dp[i-1][j],       // 不选第 i 个，对数 j 不变
    dp[i-2][j-1] + ...// 选 i 和 i-1 配对，对数 j-1 → j
)
```

- 不选第 i 个：
    
    物品数 i-1 → i，**对数 j 不变**
- 选第 i 个（必须和 i-1 配对）：
    
    物品数 i-2 → i，**对数 j-1 → j**

只有二维状态，才能同时表达：

**物品到哪了 + 已经凑了几对**

# 5. 最简单类比（秒懂版）

类比成背包问题你就瞬间明白了：

- 01 背包：
    
    `dp[i][j]` = 前 i 个物品，背包容量 j 的最大价值
- 这道题：
    
    `dp[i][j]` = 前 i 个物品，凑 j 对的最小代价

结构一模一样：

- 一个维度管**物品**
- 一个维度管**限制条件**
    
    背包是容量，这题是**对数 k**

# 6. 最终极简笔记（背诵版）

## 为什么二维？

因为要同时记录两个信息：

1. 处理到第几个物品（i）
2. 已经选了几对（j）
    
    一维存不下两个信息。

## 为什么第二维是 k？

因为题目要求**恰好选 k 对**，

DP 必须用一维专门记录**当前选了 j 对**。

## 状态记忆口诀

> **前 i 个，选 j 对，最小代价 dp [i][j]**
