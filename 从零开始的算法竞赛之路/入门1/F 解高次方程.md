代码使用二分法求解 $(8x^4 + 7x^3 + 2x^2 + 3x + 6 = Y)$ 在$ x \in [0,100] )$ 内的根。

---

## 1. 题目解析

**题目描述**  
给定一个实数 \(Y\)，求方程
\[
8x^4 + 7x^3 + 2x^2 + 3x + 6 = Y
\]
的解 \(x\)，其中 \(x\) 的取值范围为 \([0, 100]\)。如果存在解，输出保留四位小数；否则输出 `"No solution!"`。输入包含多组测试数据。

**输入格式**  
- 第一行一个整数 \(T\)，表示测试用例数量。  
- 接下来 \(T\) 行，每行一个实数 \(Y\)。

**输出格式**  
- 对于每个 \(Y\)，如果存在 \(x \in [0,100]\) 使得方程成立，则输出该 \(x\)（保留四位小数）；否则输出 `"No solution!"`。

**样例（推测）**  
```
输入：
1
100
输出：
??
```

**关键点**  
函数 \(f(x) = 8x^4 + 7x^3 + 2x^2 + 3x + 6\) 在 \([0, +\infty)\) 上单调递增，因此给定 \(Y\) 后，只需在区间内二分查找即可。

---

## 2. 数学原理

### 2.1 函数性质
\[
f(x) = 8x^4 + 7x^3 + 2x^2 + 3x + 6
\]
- **定义域**：全体实数，但题目限定 \(x \in [0,100]\)。
- **单调性**：  
  求导 \(f'(x) = 32x^3 + 21x^2 + 4x + 3\)，当 \(x \ge 0\) 时，各项均为非负且至少有一项为正，故 \(f'(x) > 0\)，因此 \(f(x)\) 在 \([0, +\infty)\) 上严格单调递增。
- **值域**：  
  \(f(0) = 6\)，\(f(100) = 8\cdot 100^4 + 7\cdot 100^3 + 2\cdot 100^2 + 3\cdot 100 + 6\)，是一个极大的数。  
  因此，当 \(Y < 6\) 时无解（不在值域内）；当 \(Y > f(100)\) 时也无解（超出上界）。但由于 \(f(100)\) 巨大，实际上只要 \(Y \ge 6\) 且在合理范围内，一般有解。

### 2.2 二分法原理
对于单调连续函数，给定目标值 \(Y\)，可通过二分查找逼近解：
- 初始区间 \([L, R]\) 满足 \(f(L) \le Y \le f(R)\)（若端点值不满足，则无解）。
- 每次取中点 \(mid = (L+R)/2\)，计算 \(f(mid)\)。
- 若 \(f(mid) < Y\)，则解在 \([mid, R]\)，令 \(L = mid\)；否则解在 \([L, mid]\)，令 \(R = mid\)。
- 重复直到区间长度小于所需精度（如 \(10^{-8}\)）。

## 3. 代码实现

```cpp
#include <iostream>
#include <cstdio>
using namespace std;

// 函数 f(x) = 8x^4 + 7x^3 + 2x^2 + 3x + 6
double f(double x) {
    return 8*x*x*x*x + 7*x*x*x + 2*x*x + 3*x + 6;
}

int main() {
    int T;
    cin >> T;
    while (T--) {
        double Y;
        cin >> Y;
        
        // 判断解是否存在：Y 必须在 [f(0), f(100)] 内
        if (Y < f(0) || Y > f(100)) {
            printf("No solution!\n");
            continue;
        }
        
        double left = 0.0, right = 100.0;
        // 二分查找，直到区间长度小于 1e-8（保证四位小数的精度）
        while (right - left > 1e-8) {
            double mid = (left + right) / 2.0;
            if (f(mid) < Y) {
                left = mid;
            } else {
                right = mid;
            }
        }
        // 输出 left 或 right 均可，精度满足要求
        printf("%.4f\n", left);
    }
    return 0;
}
```

**改进说明**：
- 原代码中初始区间为 \([-1, 101]\)，且在最后通过 `left > 0.0` 判断是否输出解，可能漏掉 \(Y = 6\) 时 \(x=0\) 的情况，且负数区间无意义。修正为 \([0, 100]\) 并提前判断解是否存在。
- 使用 `printf` 格式化输出，保留四位小数。

## 4. 编程思想

### 4.1 利用单调性简化问题
- 连续单调函数使二分法成为求解方程的利器。先分析函数在给定区间内的单调性，确保二分正确性。
- 若函数不单调，则需要更复杂的方法（如分段处理）。

### 4.2 二分法的实现细节
- **边界处理**：确保初始区间两端函数值一正一负（或一小于一大于目标值）。这里直接取左端点 \(f(0)\) 和右端点 \(f(100)\)，并检查 \(Y\) 是否在两者之间。
- **精度控制**：循环条件用 `right - left > 1e-8`，通常要求输出四位小数，这样的精度足够（2^{-27} ≈ 7.45e-9，迭代约 27 次可达 1e-8）。
- **浮点数比较**：避免直接判断相等，用区间逼近。

### 4.3 代码鲁棒性
- 输入可能包含多组数据，使用 `while(T--)` 循环处理。
- 输出格式严格遵循题目要求（四位小数，大写 "No solution!"）。

### 4.4 性能分析
- 二分法每次迭代计算一次 \(f(x)\)，复杂度 \(O(\log(\text{精度}))\)，对于每组数据约 30 次计算，非常高效。

## 总结

本题是典型的**二分法求方程根**问题，核心在于：
- 证明函数在区间上的单调性；
- 确定初始区间并验证解的存在性；
- 控制二分精度以输出所需小数位。
