## 题目解析

**题目描述**  
定义：
\[
F(0)=7,\quad F(1)=11,\quad F(n)=F(n-1)+F(n-2)\ (n\ge 2)
\]
输入多组 \(n\)（\(n < 10^6\)），判断 \(F(n)\) 是否能被 3 整除，输出 `"yes"` 或 `"no"`。

**关键**：直接计算数值会溢出，只需判断模 3 的结果。

---

## 数学原理（简要）

虽然您不打算用数学周期法，但为了笔记完整性，仍简述：
- 在模 3 下，\(F(n) \bmod 3\) 构成周期为 8 的序列。
- 当且仅当 \(n \bmod 8 = 2\) 时，\(F(n) \equiv 0 \pmod{3}\)。

但本题我们采用**记忆化搜索**，即计算 \(F(n) \bmod 3\) 并缓存结果，避免重复计算。

---

## 代码实现

### 方法一：迭代DP（自底向上记忆化）—— 推荐

这是最安全的记忆化方式，不需要递归，直接填充数组。

```cpp
#include <iostream>
#include <vector>
using namespace std;

const int MAXN = 1000000;
vector<int> memo(MAXN, -1);  // -1 表示未计算

void precompute() {
    memo[0] = 7 % 3;   // 1
    memo[1] = 11 % 3;  // 2
    for (int i = 2; i < MAXN; i++) {
        memo[i] = (memo[i-1] + memo[i-2]) % 3;
    }
}

int main() {
    precompute();  // 一次性计算所有结果
    int n;
    while (cin >> n) {
        if (memo[n] == 0)
            cout << "yes" << endl;
        else
            cout << "no" << endl;
    }
    return 0;
}
```

**说明**：  
- 用 `memo` 数组存储 \(F(i) \bmod 3\)，直接递推填充。
- 时间复杂度 \(O(\text{MAXN})\)，空间 \(O(\text{MAXN})\)，可应对所有查询。

---

### 方法二：递归+记忆化（带风险警告）

如果您坚持要用递归形式，可以这样写，但务必注意**递归深度可能达到 1e6，极易导致栈溢出**（通常栈空间只有几 MB）。在本地或某些在线评测系统中可能无法运行。以下代码仅作为演示：

```cpp
#include <iostream>
#include <vector>
using namespace std;

vector<int> memo(1000000, -1);  // 全局记忆化数组

int F_mod3(int n) {
    if (memo[n] != -1) return memo[n];
    if (n == 0) return memo[0] = 7 % 3;   // 1
    if (n == 1) return memo[1] = 11 % 3;  // 2
    return memo[n] = (F_mod3(n-1) + F_mod3(n-2)) % 3;
}

int main() {
    int n;
    while (cin >> n) {
        if (F_mod3(n) == 0)
            cout << "yes" << endl;
        else
            cout << "no" << endl;
    }
    return 0;
}
```

**风险**：当 \(n\) 较大时（如接近 1e6），递归调用深度会达到 \(n\)，极易导致**栈溢出**（`Segmentation Fault`）。竞赛环境中通常不推荐递归深度超过 \(10^5\)。  
**改进**：如果确实要用递归，可尝试增加栈空间（如 `#pragma comment(linker, "/STACK:1024000000,1024000000")` 在 Windows 下），但依然不保险。

## 编程思想

1. **记忆化搜索**  
   将计算结果缓存，避免重复计算。适用于递归结构，但需注意递归深度问题。

2. **自底向上 vs 自顶向下**  
   - **自顶向下（递归+记忆化）**：直观，但可能因深度过大而栈溢出。  
   - **自底向上（迭代DP）**：安全、高效，本质也是“记忆化”的一种实现形式。

3. **空间换时间**  
   用数组存储中间结果，换取查询时的 \(O(1)\) 时间。本题 \(n < 10^6\)，数组大小可接受。

4. **模运算简化**  
   只存储模 3 结果，避免大数溢出，且满足整除判断需求。

5. **递归深度问题**  
   在算法竞赛中，当数据规模较大时，应尽量避免深度递归，改用迭代或尾递归优化（C++ 不保证尾递归优化）。
