# P1164 小A点菜

## 题目背景

uim 神犇拿到了 uoi 的 ra（镭牌）后，立刻拉着基友小 A 到了一家……餐馆，很低端的那种。

uim 指着墙上的价目表（太低级了没有菜单），说：“随便点”。

## 题目描述

不过 uim 由于买了一些书，口袋里只剩 $M$ 元 $(0 < M \le 10000)$。

餐馆虽低端，但是菜品种类不少，有 $N$ 种 $(1 \le N \le 100)$，第 $i$ 种卖 $a_i$ 元 $(0 < a_i \le 1000)$。由于是很低端的餐馆，所以每种菜只有一份。

小 A 奉行“不把钱吃光不罢休”的原则，所以他点单一定刚好把 uim 身上所有钱花完。他想知道有多少种点菜方法。

由于小 A 肚子太饿，所以最多只能等待 $1$ 秒。

## 输入格式

第一行两个整数 $N$ 和 $M$，分别表示菜品种类和 uim 身上的钱数。

第二行 $N$ 个正整数 $a_i$（可能有重复），用空格隔开，分别表示每种菜的价格。

## 输出格式

一个正整数，表示点菜方案数，保证答案的范围在 $[0,2^{31}-1]$ 之内（不超过 C/C++的 int 范围）。

## 输入输出样例 #1

### 输入 #1

```
4 4
1 1 2 2
```

### 输出 #1

```
3
```

# 题目解析
```cpp
#include <bits/stdc++.h>  
using namespace std;  
int main(){  
    int N,M;  
    cin >> N >> M;  
    vector<int>price(N);  
    vector<int>dp(M+1,0);  
    for(int i = 0; i < N; i++){  
        cin >> price[i];  
    }  
    dp[0] = 1;  
    for(int i = 0; i < N;i++){  
        int p = price[i];  
        for(int j = M; j >= p;j--){  
            dp[j] += dp[j-p];  
        }  
    }  
    cout << dp[M] << endl;  
    return 0;  
}
```
### 为什么第一层枚举 `price` 能避免重复？

第一层循环枚举每一道菜（价格 `price`），本质是**逐个 “引入” 新菜品**，并基于 “已有菜品” 的方案去扩展新方案。比如，初始时没有任何菜品，只有 `dp[0] = 1`（什么都不选）。当引入第一道价格为 `a` 的菜时，只能在原有方案（`dp[j]`）基础上，新增 “选这道菜” 的方案（`dp[j - a]`）。当引入第二道价格为 `b` 的菜时，只能在 “已有前 1 道菜的所有方案” 基础上，新增 “选这道 b 菜” 的方案。以此类推，**每道菜只会被 “考虑一次”**，不会出现 “同一道菜被选多次” 的情况。

### 为什么第二层逆序枚举金额能保证 “不重复计算”？

假设当前处理第 `i` 道菜（价格 `p`），第二层循环从 `m` 逆序遍历到 `p`，目的是**确保使用的是 “未选当前菜” 时的旧状态**。

举个反例：如果正序遍历（从 `p` 到 `m`），会发生什么？比如，当计算 `dp[j]` 时，`dp[j - p]` 可能已经被当前菜更新过（即已经包含了 “选当前菜” 的情况），这会导致 **同一道菜被多次计入**（相当于 “无限取用”，变成了完全背包问题）。

而逆序遍历中，`j` 从大到小，`j - p` 一定小于 `j`，此时 `dp[j - p]` 还是 “未处理当前菜” 时的旧值（只包含前 `i-1` 道菜的方案）。因此，`dp[j] += dp[j - p]` 本质是：**新方案数 = 不选当前菜的旧方案数 + 选当前菜的新方案数（基于前 i-1 道菜凑出 j-p 的方案）**

### 结合样例再看：

输入：`4 4`，价格 `[1,1,2,2]`

- 初始 `dp = [1,0,0,0,0]`（只有 0 元有 1 种方案）。

1. 处理第一道菜（1 元）：逆序遍历 `j=4,3,2,1`，更新后 `dp = [1,1,0,0,0]`（新增了 “选 1 元” 凑 1 元的方案）。
    
2. 处理第二道菜（1 元）：逆序遍历 `j=4,3,2,1`，更新后 `dp = [1,2,1,0,0]`（新增了 “选第二道 1 元” 的方案：凑 1 元有 2 种，凑 2 元有 1 种（1+1））。
    
3. 处理第三道菜（2 元）：逆序遍历 `j=4,3,2`，更新后 `dp = [1,2,1,2,1]`（新增了 “选 2 元” 的方案：凑 2 元 = 原 1 种 + 1 种（直接选 2 元）=2 种？不，这里实际计算后是 `dp[2] = 1 + dp[0] = 2`，`dp[4] = 0 + dp[2] = 1` 等）。
    
4. 处理第四道菜（2 元）：最终 `dp[4]` 会累积到 3 种，即样例答案。
    

简单说，两层循环的配合实现了：**每道菜 “要么选一次，要么不选”**，既不遗漏任何可能的组合，也不会重复计算 —— 这正是 0-1 背包的精髓～
