### 🎙️ PI-Math 交互式数学计算环境 (MVP) 汇报大纲

#### 1. 项目概述与目前功能 (What we have done)
本项目旨在打造一个**类 MATLAB 的交互式数学计算与算法验证环境**，目前已完成 MVP（最小可行性产品）阶段。具备以下核心功能：
* **多形态客户端支持**：支持纯 Web 端访问（Vite 构建）以及跨平台桌面客户端（基于 Electron），满足开箱即用的本地需求。
* **REPL 交互式终端**：提供了类似命令行的前端交互界面，用户可以输入单行指令或批量脚本，立即得到执行结果。
* **可视化工作区 (Workspace)**：能够像 MATLAB 一样，实时捕获并抓取当前执行环境中的局部变量，在右侧面板动态展示变量的**名称、类型及当前值**。
* **原生级文件管理**：集成了操作系统的原生文件选择与保存对话框。桌面端使用 Electron IPC 穿透，Web 端使用前沿的 File System Access API 实现无缝的目录树读取和新建/覆盖代码文件。
* **专业级代码编辑器**：集成了微软 Monaco Editor，并深度定制了“仓颉 (Cangjie)”语言的语法高亮和括号匹配。

#### 2. 它能做到的边界与局限性 (Boundaries & Limitations)
目前系统的架构决定了我们在现阶段的局限性，这也是 MVP 验证后需要突破的边界：
* **执行性能边界**：当前的交互执行依赖于“前端提交代码 -> 后端生成临时脚本 -> `cjpm run` 编译并启动子进程 -> 读写 `workspace.json` 捕获状态”的架构。对于高频的小计算而言，频繁的子进程拉起和编译冷启动会带来巨大的 I/O 延迟开销，不适合毫秒级的实时级绘图推流。
* **状态持久化边界**：目前采用的是“脚本拼装+变量序列化注入”的伪常驻内存方案。遇到巨型矩阵、超大浮点数数组时，通过 JSON 序列化转存环境状态会触及内存与解析瓶颈。
* **安全与沙盒边界**：目前直接调用系统进程执行仓颉代码，拥有极高的系统权限。如果未来要作为 Web 云服务发布，极易遭遇恶意代码注入（RCE），目前仅适合本地安全的桌面环境运行。

#### 3. 为什么选择这套技术栈？(Tech Stack Justification)
* **核心计算层：仓颉 (Cangjie)**
  * **选型原因**：作为新一代强类型、支持高性能并发的底层语言，不仅契合未来的国产化/信创生态生态，并且语法现代，适合构建严谨的科学计算库 (`PiMath`)；用它作为数学执行引擎，兼具执行速度与安全性探索。
* **前端展示层：Vite + Monaco Editor + 原生 JS/CSS**
  * **选型原因**：Vite 提供了极速的冷启动和热更新，极大加快了我们的开发效率；选用 Monaco Editor 是因为它就是 VS Code 的内核，可以提供最完美的编辑体验（语法高亮），为之后接入代码提示（LSP）打下基础。
* **跨端外壳层：Electron + 独立 Node.js Server**
  * **选型原因**：由于我们需要直接执行本地的 `cjpm`（仓颉包管理器）和读写本地项目文件，纯浏览器无法跨越安全沙盒。Electron 完美弥补了这一空缺，让我们可以使用前端技术零成本打包出一个独立桌面 App。

#### 4. 未来的简单展望 (Future Outlook)
* **增强数学/绘图核心库**：丰富 `PiMath` 原生计算库，完善对复数、线性代数（矩阵）、微积分的支持。完善目前初步集成的 `gnuplot` 绘图联动，实现类似 MATLAB `plot()` 的窗口弹图体验。
* **工具箱生态 (Toolbox)**：提供简单的宏 (Macro) 和插件机制，允许用户导入特定的算法包直接在环境里交互式调用。
* **智能化 (AI 赋能)**：未来可以基于收集到的报错日志、执行痕迹，接入大模型 API 提供自然语言生成计算脚本的功能。

#### 5. 架构优化建议 (Optimization Suggestions)
* **从“文件读写”转向“进程通信 (RPC / TCP / WebSocket)”**：
  建议废弃每次运行都要读写 `workspace.json` 并重新 `spawnSync` 拉起子进程的做法。应在后台启动一个**常驻的仓颉守护进程 (Daemon)**，前端代码通过 TCP 或 WebSocket 将指令发送给该进程并在内存中直接 eval / 解释执行，结果直接通过 Socket 推送回来。这将带来百倍的提速。
* **接入 LSP (Language Server Protocol)**：
  未来在 Monaco Editor 中接入仓颉语言服务 LSP 服务，可以在前端直接实现报错红波浪线、代码自动补全、函数参数悬浮提示等工业级 IDE 功能。
* **沙盒隔离 (WebAssembly)**：
  如果条件成熟，考虑将仓颉的计算核心编译为 WebAssembly运行在浏览器内存中，彻底消灭本地环境依赖问题，让新用户打开网页就能即刻使用 PI-Math。
