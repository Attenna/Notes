# pi_math_mvp 项目配置文档（适配仓颉 0.55.18 + pimath）

## 文档说明

本文档为 `pi_math_mvp` 项目的核心配置手册，统一约定仓颉环境、项目配置、依赖管理规则，适配仓颉 0.55.18 版本（匹配 pimath 开发版本），确保团队成员环境一致、开发无兼容问题。

这里给出了Windows 11 家庭版的参考部署流程。各位可以参考
## 一、环境配置（全员统一）

### 1. 必装软件 & 版本（核心，请勿随意更换）


| 软件 / 组件       | 版本要求                | 作用                | 验证方式（终端执行）                               |
| ------------- | ------------------- | ----------------- | ---------------------------------------- |
| 仓颉 SDK        | 0.53.18（离线安装包）      | 项目运行 / 编译核心环境     | `cj -v` → 输出 `0.53.18`                   |

## 二、项目核心配置文件（根目录下创建）

### 1. cjpm.json（包管理配置）

```json
{
  "name": "pi_math_mvp",
  "version": "0.1.0",
  "description": "轻量化MATLAB仿品（MVP），基于仓颉0.55.18+libPimath",
  "author": "你的团队名",
  "type": "module",
  "dependencies": {
    "pimath": "file:./lib/pimath.cjo"
  },
  "scripts": {
    "start": "cj run src/main.cj",
    "test": "cj run test/test_basic.cj"
  },
  "cjpm": {
    "sdkVersion": "0.55.18"
  }
}
```

**关键说明**：

- `dependencies` 仅声明本地依赖 `pimath.cjo`，无需远程依赖；
- `scripts` 为快捷命令，全员统一用 `npm run start` 启动项目，避免记复杂指令；
- `sdkVersion` 必须和仓颉安装版本一致，禁止修改。

### 2. cjconfig.json（运行 / 编译配置）

```json
{
  "sdk": {
    "version": "0.55.18",
    "runtime": "nodejs"
  },
  "build": {
    "target": "esmodule",
    "outDir": "./dist",
    "sourceMap": false
  },
  "run": {
    "entry": "./src/main.cj"
  }
}
```

**关键说明**：

- `runtime: "nodejs"`：适配跨平台运行，无需鸿蒙环境；
- `sourceMap: false`：关闭源码映射，避免早期版本兼容问题；
- `entry`：指定项目入口文件，和 `scripts.start` 保持一致。

### 3. .cjignore（忽略文件配置）
```plaintext
# 运行时数据目录（自动生成，无需编译）
data/
# 编译输出目录
dist/
# 前端目录（仓颉不编译HTML/JS）
web/
# 文档目录
docs/
# 临时文件
*.log
# 系统隐藏文件
.DS_Store
Thumbs.db
# 无关依赖
node_modules/
```

**关键说明**：避免仓颉编译 / 打包时包含无关文件，减少冗余。

## 三、依赖验证（全员执行）

在项目根目录 `pi_math_mvp/` 下执行以下命令，验证所有依赖配置正常：

```bash
# 初始化cjpm（仅首次执行）
cjpm init -y
# 安装本地依赖（验证pimath路径）
cjpm install
# 验证pimath调用（临时测试，可选）
echo "import pimath from './lib/pimath.cjo'; let x = pimath.linspace(1,10,10); console.log(x);" > test_temp.cj
cj run test_temp.cj
# 验证成功后删除临时文件
rm test_temp.cj
# 启动项目（最终验证）
npm run start
```

**验证标准**：

- 无 `module not found` 报错；
- 终端输出 pimath 生成的数组 `[1,2,...,10]`；
- `npm run start` 能正常启动，无版本兼容报错。

## 四、版本专属适配规则（仓颉 0.55.18）

### 1. 代码编写注意事项

|功能点|正确写法（适配 0.55.18）|错误写法（禁止使用）|
|---|---|---|
|模块导入|`import pimath from "../../lib/pimath.cjo";`（默认导入）|`import * as pimath from "../../lib/pimath.cjo";`（通配导入）|
|JSON 读写|补充 try-catch 捕获解析错误（示例如下）|无错误捕获，直接 JSON.parse/stringify|
|eval 执行|显式注入全局变量：`globalThis.globalWS = globalWS;`|直接 eval 执行，未注入全局变量|

#### JSON 读写适配示例（核心）

```cangjie
import { readFile, writeFile } from "fs";

// 写JSON文件
function writeWorkspaceJson(data) {
  try {
    let jsonStr = JSON.stringify(data, null, 2);
    writeFile("./data/workspace.json", jsonStr);
  } catch (e) {
    console.log("JSON写入失败：", e.message);
  }
}

// 读JSON文件
function readWorkspaceJson() {
  try {
    let jsonStr = readFile("./data/workspace.json");
    return JSON.parse(jsonStr);
  } catch (e) {
    // 兜底返回空配置
    return { user_script: "", workspace_vars: {}, error_msg: "", run_flag: false };
  }
}
```

gnuplot 调用命令（gnuplot_plot.cj）：

## 五、常见问题 & 解决方案

|问题现象|解决方案|
|---|---|
|提示 “找不到 libPimath.dll”|1. 检查 `lib/` 目录是否有对应库文件；2. 确认库文件适配仓颉 0.55.18 + 系统位数|
|JSON.parse 报错 “unexpected token”|1. 检查 JSON 字符串无末尾逗号等语法错误；2. 补充 try-catch（参考四.1 示例）|
|eval 执行提示 “undefined function”|执行前注入全局变量：`globalThis.globalWS = globalWS;`|
|cjpm install 失败|1. 检查 cjpm 版本为 0.55.18；2. 确认 `cjpm.json` 中 pimath 路径正确|
|绘图窗口不弹出|1. 验证 gnuplot 已配置环境变量；2. 检查 system 命令路径是否正确|
