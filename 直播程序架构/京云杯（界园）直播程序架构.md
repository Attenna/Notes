## res://
### main
`main.tscn`
`main.gd`
main承担了主窗口的工作，所有的直播用场景都在这里加载。

### ctrl_panel
`ctrl_panel.tscn`
`ctrl_panel.gd`
承担了控制面板的工作，其实也有事件总线的意味
由main启动
启动时，优先加载addons里面的内容，并且把addons里面的交互空间根据一定顺序（优先级）进行排版。
决定了程序的关闭

## user://
### operators
干员数据，由python爬虫从PRTS爬取，并且同目录下有存储干员名称的json文件。
### squads
编队数据，由python爬虫从PRTS爬取，并且同目录下有存储编队名称的json文件。
## endings
结局数据，结算页面使用，数据来自PRTS，并且同目录下有存储结局名称的json文件。
### players
记录选手信息的文件夹，由两个文件，一个文件夹构成：
`players.json`
`now_player.json`

players的格式可以很自由，毕竟json是个字典。你可以有选手的名字，头像的目录，分数，排名，还有他的赛前感言……，插件可以完全访问，修改这些数据。
```json
{
	"player1":{
		"name":"George",
		"score":"325",
		"icon_path":"user://players/player_icons/George.jpg",
		"slogan":"325325"
	},
	"player2":
	{
		"name":"Genshin",
		"score":"648",
		"icon_path":"user://players/player_icons/Genshin.jpg",
		"slogan":"我要玩元神"
	}
}
```
#### player_icons
子文件夹，存储了一一对应的选手头像
#### team_icons
子文件夹，存储了选手对应的战队信息，不过鉴于比赛没有战队，于是我不写。

### scenes
场景文件，由main启动的时候读取目录内容，不同的scene和对应的gd脚本都保存在里面。
内置一个json文件，json记录了有哪些场景
```json
{
	"主场景":"main",
	"转场":"clip",
	"比赛场地":"streaming_arena"
}
```
### addons
用于存储插件的文件夹。插件的定义：
```json
{
	"转场控制":"slides",
	"选手登记":"player_regi",
	"计算器":"calc",
	"联网远程控制":"web_remote"
}
```
